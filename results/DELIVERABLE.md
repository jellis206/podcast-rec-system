## Technical Notes

- **L2 (Euclidean) Distance**: Used via pgvector's `<->` operator
- **Lower values** indicate higher similarity
- All query documents are excluded from their own result sets

### Database Implementation
- **Vector Index**: IVFFlat with 100 lists for approximate nearest neighbor search
- **Episode Averaging**: SQL `AVG(embedding)` function computes element-wise mean of 128-dimensional vectors
- **Foreign Keys**: Segment table references podcast table for data integrity
